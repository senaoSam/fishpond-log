# 魚塭紀錄 App — 規格文件 (SPEC)

> 最後更新:2026-06-23

## 1. 目標

取代紙筆手寫的魚塭餵食/用藥紀錄,改用手機隨手記錄、電腦彙整查看。

- **平常**:在魚塭現場用手機記錄(取代紙筆)。
- **週末 / 每月**:用電腦看整月統計、匯出 Excel。
- **數月後**:可查詢任一時間的歷史紀錄,一目了然。

只有使用者一人使用,資料為純文字,不涉及敏感資安問題。

---

## 2. 技術決策(已討論確認)

| 項目 | 決策 | 理由 |
|---|---|---|
| 形式 | **PWA**(漸進式網頁 App) | 免費、免上架、可「加到主畫面」像 App、網頁手機共用一份程式碼 |
| 部署 | **GitHub Pages**(純靜態) | 免費 |
| 資料庫 | **Firebase Firestore**(Spark 免費方案,不綁卡) | 即時同步:手機記完電腦馬上看到;免費額度遠超個人用量 |
| 離線 | 不強制(現場通常有網路) | 仍會啟用 Firestore 內建離線快取,斷線時可暫存 |
| 前端 | 純 HTML + CSS + JavaScript(無框架 / 或輕量框架) | 靜態部署最單純、維護容易 |
| Excel 匯出 | 前端用 [SheetJS (xlsx)] 產生 `.xlsx` | 不需後端 |

## 3. 資料模型 (Firestore)

### Collection: `records`(每筆紀錄一份文件)

| 欄位 | 型別 | 說明 | 範例 |
|---|---|---|---|
| `pond` | string | 池塘名稱(變動,可自由輸入或從清單選) | "1號池" |
| `date` | string (YYYY-MM-DD) | 日期 | "2026-06-23" |
| `period` | string | 時段:`morning`(早上)/ `afternoon`(下午) | "morning" |
| `bags` | number | 輸入幾包(變動) | 3 |
| `feedNo` | string | 飼料編號(固定選項,可新增) | "2號" |
| `mix` | string | 拌料(固定選項,可新增,**可不填**) | "安蒙20%" |
| `disinfectant` | string | 消毒劑(固定選項,可新增,**可不填**) | "二氧化氯50%" |
| `note` | string | 備註(選填) | "" |
| `createdAt` | timestamp | 建立時間(自動) | — |

> **早上/下午**:一天兩筆獨立記錄。同一池塘同一天可有 morning + afternoon 兩筆。

### Document: `settings/options`(可編輯的固定選項清單)

```json
{
  "ponds":         ["1號池", "2號池"],              // 池塘名稱建議清單(可自由新增/輸入)
  "feedNos":       ["0.8號","000號","00號","0號","1號","2號","3號","4號","5號","6號","7號"],
  "mixes":         ["安蒙20%","安蒙50%","弗洛得","OTC"],
  "disinfectants": ["二氧化氯10%","二氧化氯50%","三氯20G","三氯90%"]
}
```

- 這四個清單都「可編輯新增」。在設定頁可增刪。
- 新增的選項即時寫回 Firestore,所有裝置同步。

## 4. 預設固定選項(初始值)

| 欄位 | 預設選項 |
|---|---|
| 飼料編號 | 0.8號、000號、00號、0號、1號、2號、3號、4號、5號、6號、7號 |
| 拌料 | 安蒙20%、安蒙50%、弗洛得、OTC |
| 消毒劑 | 二氧化氯10%、二氧化氯50%、三氯20G、三氯90% |

> 飼料編號排序註記:目前依你給的順序為 0.8 → 000 → 00 → 0 → 1...7。實作時保留此順序,如要調整可在設定頁排序。

## 5. 功能 / 頁面

### 5.1 記錄頁(手機主用)
- 大按鈕、大欄位,適合現場單手操作。
- 欄位順序:
  1. **池塘名稱** — 下拉選 + 可手動輸入新名稱
  2. **日期** — 預設今天,可改
  3. **時段** — 早上 / 下午(兩顆切換按鈕)
  4. **輸入幾包** — 數字鍵盤
  5. **飼料編號** — 下拉(含「+ 新增」)
  6. **拌料** — 下拉(含「+ 新增」,可不選)
  7. **消毒劑** — 下拉(含「+ 新增」,可不選)
  8. **備註** — 選填
- 「儲存」後清空表單、保留池塘/日期方便連續記下一池。

### 5.2 紀錄列表 / 查詢頁
- 依日期區間、池塘篩選。
- 預設顯示本月,可切換月份。
- 每筆可編輯 / 刪除。
- 清楚列出每筆:日期、時段、池塘、包數、飼料、拌料、消毒劑。

### 5.3 月統計頁(電腦主用)
- 選擇月份。
- 統計內容(初版,已確認):
  - 當月各池塘**總包數**。
  - 各**飼料編號個別包數**加總。
- (之後可再加:拌料/消毒劑次數、每日早晚明細、圖表)
- 一目了然的表格 + 簡單合計列。

### 5.4 匯出 Excel
- 在統計頁 / 列表頁可「匯出 Excel」。
- 匯出當前篩選結果(整月或區間)為 `.xlsx`。
- 欄位:日期、時段、池塘、包數、飼料編號、拌料、消毒劑、備註。
- 檔名範例:`魚塭紀錄_2026-06.xlsx`。

### 5.5 設定頁
- 編輯四組固定選項(池塘、飼料編號、拌料、消毒劑):新增、刪除、排序。

## 6. 安全性

**決策:完全公開,不加閘門。**

- Firestore 規則設為允許讀寫(test mode)。
- 使用者一人、純文字、機率低,接受此風險。
- 日後若要,可再加密碼頁或匿名登入。

---

## 7. 不做的事(範圍界線)

- 不做多使用者 / 權限管理。
- 不做複雜離線同步衝突處理(僅用 Firestore 內建快取)。
- 不做圖表(初版用表格;之後想要再加)。
- 不做原生 App 上架。

---

## 8. 實作步驟(預定)

1. 建立 Firebase 專案 + Firestore(我會給你逐步設定指引)。
2. 專案骨架:`index.html` + `app.js` + `style.css` + `manifest.json` + `service-worker.js`(PWA)。
3. 串接 Firestore:讀寫 `records` 與 `settings/options`。
4. 記錄頁 → 列表頁 → 統計頁 → 匯出 Excel → 設定頁,逐頁完成。
5. PWA 設定(可加到主畫面、icon)。
6. 部署到 GitHub Pages。
